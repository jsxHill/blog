利用yeoman生成项目代码

yeoman是什么

yeoman是一款使用Node开发的项目代码生成工具，支持多种语言（Web, Java, Python, C#, 等等)。

安装&使用generator

自定义generator

yeoman本身不生成任何代码，而是把这项工作交给生成器插件。

创建一个generator

generator本质上是一个node_module，目录结构如下：

generators为生成器目录。app是默认生成器，供命令yo name使用。
router是子生成器，供命令yo name:sub-generator使用。

package.json配置如下：

  1. name必须以generator-开头。
  2. keywords必须包含yeoman-generator。
  3. 发布后如果期望generator能够被yeoman官方收录，description是必填项。
  4. yeoman-generator是生成器的基类，后续都是在其基础上进行扩展开发。
  5. files定义了生成器所在目录（配合yo命令使用）。

task & run loop

generator由一系列task（即generator原型对象上的method）组成，并通过run loop队列执行task。
另外yeoman还定义了task的优先级（通过name匹配），特殊优先级名称及其执行顺序如下：

  1. initializing，初始化方法
  2. prompting，处理与用户交互的部分
  3. configuring，修改以及保存项目配置
  4. default，特殊优先级之外都归入此类
  5. writing，文件写入处理
  6. conflicts，冲突处理
  7. install，依赖安装（npm、bower）
  8. end，结束/清理阶段

使用问题总结

this.sourceRoot(path)更改模版存放路径后导致模板文件找不到

默认情况下，模板文件目录是generator下的templates目录（跟index.js同级）。比较坑的是，如果使用sourceRoot(path)修改root，其上下文是destination 目录而非generator目录。因此，这个方法应当避免使用（官方已视其为遗留方法）。
获取generator根目录可以使用this.resolved，获取模板文件path可以使用this.templatePath()。

.gitignore无法复制

本地运行好好的generator发布到npm后再使用，结果发现.gitignore文件没有复制，还多了一份.npmignore文件。原来这是npm导致的。npm publish会自动过滤掉.gitignore文件，因此.gitignore根本不会提交到仓库，自然也无法复制。另外如果未提供.npmignore文件，但存在.gitignore文件，npm会copy .gitignore并作为.npmigore上传，因此多了一份.npmignore文件。
解决办法是，将.gitignore重命名为gitignore_copy，然后在generator中处理为.gitignore。
