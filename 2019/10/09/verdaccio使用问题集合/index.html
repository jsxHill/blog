verdaccio使用问题集合

npm audit ‘Cannot convert undefined or null to object’

enviroment:
MAC os10.14
Node v8.14.0
npm v6.4.1

errInfo:

reason:
切换npm前遗留的packcage-lock.json引起，registry指向不对。

solution:
移除packcage-lock.json，node_modules重新安装。

Cannot publish over existing version

即使更新了package version，仍然会报这个错误。

reason:
私有package与公有package重名。发布时，verdaccio误读了公有package的version，若该version大于本地version，则会出错。

solution:
npm publish –force 私有强制覆盖公有。
或者修改私有package的名称，避免冲突。

maximum amount of users reached

reproduce:
按照官方文档说明，启用htpasswd，并设置max_users为-1。

预期行为是禁止客户端注册，由管理员手动向htpasswd添加用户。结果却是已注册的用户也无法登录。

enviroment:
MAC os10.14
verdaccio v3.x

reason:
verdaccio自身的问题（至少3.x版本存在）。

solution:
任选其一

  1. 在verdaccio web站点上登录一次之后，即可通过npm cli登录。
  2. 将max_users设为1，让verdaccio允许登录，同时无法在客户端注册（因为我们至少内置了一个账号）。

zlib: unexpected end of file

reproduce:
通过npm（指向verdaccio）安装ivux（私服包），始终报错：

enviroment:
MAC os10.14
verdaccio v3.x

solution:

  1. 清除verdaccio缓存(storage)、本地npm缓存(npm clean –force)、package.lock.josn文件、node_modules文件夹均无效。
  2. 服务器重装verdaccio也无效。
    本地机搭建verdaccio服务没有问题。在服务器上创建npm项目也没有问题。
    基本确定问题是出在服务器与客户机通信上。似乎是网络下载异常导致的，体积稍微大一点的包就容易截断（ivux package > 750kb）。
  3. 最后发现是服务器上的ng被改了，还原ng并重设解决问题。
