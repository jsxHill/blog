scrapy入门

安装

scrapy是最受欢迎的python爬虫框架之一，很适合初学者学习。
话不多说，本地已有anaconda以及conda-forge，直接安装scrapy。

安装成功之后，来实践一下官方的爬虫例子。

运行成功后，可以看见爬取下来的html文件（不带css样式）。

scrapy tool（CLI）

scrapy命令分2类：

  1. 全局命令（Global commands），可以在任意目录下执行；
  2. 项目命令（Project-only commands），需要在某个project目录下才能执行。

项目结构

scrapy项目结构：

Configuration 项目配置

自定义scrapy tool（CLI）行为，支持多层级配置（由高到低）：

  1. 项目配置文件 scrapy.cfg
  2. 用户级配置文件 /.config/scrapy.cfg或/.scrapy.cfg
  3. 系统级配置文件 /etc/scrapy.cfg或c:\scrapy\scrapy.cfg

Setting 组件配置

自定义scrapy组件（包括core、spider、pipeline、extensions等等）行为，支持多层级配置（优先级从高到低）：

  1. 命令行参数 cmd options
  2. Spider配置对象 spider.custom_setting
  3. 项目配置文件 setting.py
  4. CLI-Command默认配置，command.default_setting
  5. 全局配置文件，scrapy.settings.default_settings

核心对象

Spider

scrapy.Spider用于定义如何抓取一个网站的内容。基本流程为：
rules filter -> start requests -> response parse -> extract data -> follow
我们需要做的，就是为每个部分填充具体的业务逻辑。

通过命令行为Spider传递参数

由-a 传递的参数会传递给Spider的__init__方法，并成为spider对象的属性。

Selector

Selector用于使用XPath或者CssSelector从html字符串中查询数据。

Selector内部依赖于parsel，而parsel依赖于lxml。与原库相比，Selector功能更加丰富：

  1. 扩展了Css选择器，支持::attr()匹配属性值，::text匹配文本节点
  2. 扩展了XPath，支持正则式、Set
  3. 支持正则式筛选处理搜索结果，如selector.xpath(“path”).re(r’xxx’)
  4. 链式混用xpath与css，充分发挥2者的优势，如selector.css(“.cls”).xpath(“//a/@href”).get()

Item

Item用于定义ExtractData的数据结构，让提取逻辑更加清晰。
开发者一般会使用自由度较高的dict来保存提取结果。但dict对象自身结构不受限制，可以任意修改。而且除了翻阅源代码，只能通过文档方式来描述现有dict的结构，在复杂场景下，这往往会引起代码可维护性问题。Item正好提供了一个集中描述结构化数据的地方。

Item是装载数据的容器，Item Loader则是专门为容器填充数据的工具。Item本身只能通过item[key]=value形式去操作数据，ItemLoader则允许你定义更加丰富的行为。
比如：

  1. 支持为field添加多个数据来源，组合输出。
  2. 支持input-processor，可以进行简单的数据格式化。
  3. 支持output-processor，自定义数据来源输出结果。

Item Pipline
ItemPipline用于流式处理Item。可以用来：

  1. 数据清洗
  2. 数据校验
  3. 数据筛选
  4. 数据存储
    ps: 可以通过no return或者raise DropItem(“reason”)丢弃Item，被丢弃的Item也不会本后续的流处理。

Feed Exports

Feed Exports用于以文件形式导出数据。支持JSON、JSON Lines、CSV、XML等多种文件格式，以及本地文件、FTP、S3等多种存储方式。
简单场景下，可以通过命令行参数 -o 直接导出数据。复杂场景下，可以自定义ItemExport，结合ItemPipline流程序列化存储数据。

Requests and Responses

Requests-Responsesrequest用于发送http请求，responses用于解析html文档。

Link Extractors

Link Extractors用于导出页面中的link对象，便于实现链接追踪。
