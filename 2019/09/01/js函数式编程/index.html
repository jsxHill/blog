js函数式编程

核心概念

-> Programming
-> lambda calculus lambda计算（自上而下的编程方式，函数式）
-> 函数匿名
-> 只接受一元参数
-> 函数为一等公民
-> Turing machine 图灵机（自下而上的编程方式，指令式）
-> 两者在数学上是等价的，任何图灵机能实现的程序，都可以转化为lambda计算

-> Functino Programming 函数式编程
-> non-determinism（不确定性） = parallel processing（并行程序） + mutable state（修改状态）
-> Pure Function（纯函数）：
-> 只用于Mapping（映射）计算的函数
-> 无副作用（一个input恒对应一个output，也不会修改或依赖任何外部变量）
-> Immutability 不可变性
-> 不修改任何外部变量
-> 不共享任何state
-> 有一个纯函数特例noop（无返回值）

-> Function Composition 函数组合
-> f∘g <=> (f∘g)(x) <=> f(g(x))
-> 函数组合是fp的核心
-> curry（柯里化）：
-> 一次只处理一个参数，然后返回处理下一个参数的函数，直到最后一个参数为止
-> 柯里化的函数是抽象的（Abstraction），偏函数是它的实例
-> 3种fp：
-> Functions map: a => b
-> Functors map with context: Functor(a) => Functor(b)
-> Monads flatten and map with context: Monad(Monad(a)) => Monad(b)

-> Functor Map 函子运算
-> functor是一种抽象模型
-> mappable，可通过map接口将范畴A映射为范畴B
-> Identity，恒等，f.map(x => x)的结构在含义上应该与f相等
-> Composition，组合，f.map(x => h(g(x)))等价于f.map(g).map(h)
-> 利用函子的组合特性，可方便实现函数组合的链式调用
-> functor的意义在于提供一种统一的方式去处理任意数据

-> Monad Map Monad运算（源自数学之范畴学）
-> monad是一种增强的functor，对外提供flatMap功能
-> flatMap
flatten M(a) -> a
f a -> b
lift M(b)
-> monad的目的在于为value添加上下文信息（Context）
-> 空值处理
-> 异常处理
-> 异步处理

-> MOP 面向组件编程，用数学替代inheritance
-> OO的本质
-> Encapsulation，封装状态
-> Message passing，消息通信
-> Dynamic binding，动态绑定
-> Object Composition 对象组合
-> 组合优于继承，继承是一种耦合度很高的对象复用方式，is-a会带来诸多问题：
-> 脆弱的基类——因为会影响诸多class，导致基类很难修改
-> 大猩猩香蕉问题——继承可能会带来一堆不需要的属性或者方法
-> 必然的重复——不同子类可能会包含相同实现，因为并非所有接口实现都不同
-> 常见的对象组合方式：
-> Aggregation 聚集，由一组可枚举的对象组成
-> Concatenation 串联，向一个对象上动态添加新属性
-> Delegation 委托，对象将访问转发给另一个对象
-> Injection 注入，对象作为参数传递给其它对象

-> Functional vs Class
-> 优先使用函数
-> 对于js而言，class是最难使用组合模式的
-> 组合复杂度：Functions > objects > factory functions > functional mixins > classes
-> class extends会带来is-a问题
-> 避免使用new
-> new class会被迫与Constructor函数耦合，重构代码变得困难
-> new class会隐式告诉别人这是一个构造函数，构造函数可以extends，而extends会导致脆弱的代码结构出现
-> 对外暴露factory function，而非class
-> class只能通过new来创建对象
-> Class适用场景
-> 封装底层通用框架
-> 此时业务比较封闭，可以设计出健康的层次结构
-> 统一创建组件实例以及管理其生命周期，对外隐藏组件class
-> 注意class最多extends一层
-> 微观性能优化
-> 利用prototype节约内存
-> 大多数场景可以忽略不计。因此只在真的有必要时考虑prototype。
-> 比如react、vue等UI组件库
-> 比如three.js等图像引擎，他们对实时渲染性能要求较高

-> Lens 一种对属性取值以设值的抽象模型
-> lens提供2个纯函数来完成取值以及设值
-> 范式
-> view(lens, set(lens, a, store)) ≡ a
-> set(lens, b, set(lens, a, store)) ≡ set(lens, b, store)
-> set(lens, view(lens, store), store) ≡ store

-> Transducers 转换器
-> Transducers是高阶reducer（缩减器）
-> 函数组合reducer
-> 高效的流式数据处理
-> 能够处理任意结构的可枚举对象
-> 无论是lazy（懒求值）或者eager（及早求值）都不影响Transducers
-> 签名
-> reducer = (accumulator, current) => accumulator
transducer = reducer => reducer
即
transducer = ((accumulator, current) => accumulator) => ((accumulator, current) => accumulator)
-> reducer是用于将多个输入变为一个输入的函数
-> Top to bottom / Left to right
-> Function Compose f∘g∘h => reducer(h) -> reducer(g) -> reducer(f)
-> rules
-> Initialization，如果accumulator没有初始值，那么transducer必须调用step为其生成一个empty值（valid initial accumulator）
-> Early termination，当接收到一个reduced时，transducer必须检查并停止pipe（提前终止reducing）
-> Completion(optional)，当缩减过程结束时，可以选择性执行一个Complete操作（after reducing，do sth）
-> fp编程环境
-> 使用js框架：Ramda RxJS Mori
-> 与underscore、lodash相比，前者是js版函数式编程，而后者是命令式的工具方法（如今很多方法都可以用原生方法替代）
-> Clojure（Script）从Lisp发展而来的函数式编程语言
-> ClojureScript可以将Clojure编译为js。如此一来，Clojure代码可以调用js函数以及在browser/node中运行
