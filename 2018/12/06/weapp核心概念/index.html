weapp核心概念

项目结构

小程序包含一个描述整体程序的app和多个描述各自页面的page。

模块化

小程序采用了CommonJS规范，通过module.exports与require的方式定义/引用模块。

页面生命周期

路由

小程序的路由全部由框架进行管理，即页面栈。对于js逻辑而言，页面栈是只读的。

路由方式 页面栈表现 触发方式
初始化 新页面入栈
打开新页面 新页面入栈 wx.navigateTo
<navigator open-type="navigateTo"/>
页面重定向 当前页面出栈，新页面入栈 wx.redirectTo
<navigator open-type="navigateTo"/>
页面返回 页面不断出栈，直到目标返回页 wx.navigateBack
<navigator open-type="navigateBack"/>
Tab 切换 页面全部出栈，只留下新的 Tab 页面 wx.switchTab
<navigator open-type="switchTab"/>
重启动 页面全部出栈，只留下新的页面 wx.reLaunch
<navigator open-type="reLaunch"/>

WXML

wxml是小程序用来描述页面结构的标记语言，类似HTML，区别在于wxml提供的是语义、功能更加明确的组件标签，并提供模版渲染语法。

ps: <block>仅仅是一个包装元素，类似vue中的<template>，不会被渲染，可接收指令。

Mustache语法

小程序template中使用{{ expression }}来绑定数据，但是expression只支持简单的js表达式:

  1. 绑定变量 {{ variablName }}
  2. 绑定true/false {{ true }}、{{ false }}
  3. 三元运算 {{flag ? true : false}}
  4. 算数运算 {{a + b}}
  5. 逻辑判断 {{a > 1}}
  6. 字符串拼接 {{a + "suffix"}}
  7. 访问对象属性 {{obj.name}}、{{arr[0].checked}}
  8. 创建新数组 {{[name, "john", "smith"]}}，支持展开运算符 {{[...fruits, "apple", "orange"]}}
  9. 创建新对象 {{title: "用户管理", code: "01"}}，支持展开运算符 {{...attrs, title: "系统设置"}}

注意:

  1. 在属性中使用Mustache需要使用””引起来。另外花括号与”之间不要有空格，"{{a}} "会被解析为"{{a + ' '}}"，可能会造成意料之外的副作用。
  2. expression中不支持函数调用，即使是原生方法也不行，如arr.indexOf(“xx”) > 0
