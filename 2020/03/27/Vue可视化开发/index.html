Vue可视化开发

前言

使用Vue开发数个项目之后，很明显感受到，在大多数情况下，我们的工作只是堆叠组件从而实现一些相似的页面。所以为什么不能通过几个简单操作生成它们？毕竟_偷懒_DRY可是编程的基本原则呀。查阅资料得知代码自动生成有2种实现思路，第一种是类似vue-cli脚手架，通过cli命令 + 模版引擎来生成。第二种是提供可视化的编辑器来生成页面。前者实现起来比较简单，但是无法应付复杂的业务代码生成场景（占用80%的开发量）。为了能够真正节约宝贵的开发人力，我们的最终目标是采取可视化编辑器的方式，实现一个允许我们引用不同组件、自定义布局以及事件绑定的前端页面自动化生成工具。

实现原理

以组件作为最小单位，通过可视化的方式组合组件从而成为一棵组件树，然后通过解析组件树生成最终的前端页面代码。

有用的工具

HTML解析/序列化

cheerio提供了在数据层面上，以类似jQuery的方式来操作DOM的能力，工程中主要用来生成HTML代码。
cheerio底层使用htmlparser2或者parse5来解析html字符串。默认逻辑是，如果xmlMode选项为true，则使用htmlparser2，否则使用parse5。另外，可以传递_useHtmlParser2: true选项来强制使用htmlparser2（注意，cheerio跟htmlparser2相关的配置项是受选择解析器影响的）。
然后cheerio底层使用dom-serializer来做序列化操作，需要注意的是其自闭和标签的处理，如果xmlMode为true，那么没有子元素的标签会自闭和。如果xmlMode为false，那么会跟html标准规范保持一致，属于自闭和标签的元素会得到处理。

js解析/序列化

recast提供了对js代码语法解析、以及反向生成js代码的能力。
recast底层使用esprima来完成js代码的语法解析（parser是可配置的），调试时可以借助esprima-demo来查看AST结构。

代办项

  1. 替换Class关键字，改为利用构造函数程式化的创建类。Class虽好，但是无法简单处理多继承场景，也无法处理mixin实例属性问题，维护静态属性也比较困难。
  2. 移除BaseComponent.js，将BoxItem与DrawItem分开使用。实际证明该组件毫无用处。
  3. 利用EnderJs封装DOM操作接口。
  4. 想办法去除多余的组件，它是因为需要利用render函数动态渲染而引入的wrap组件（函数组件不支持单独创建实例）。
    有解决方案了，直接将VueNodeTree作为数据，然后根据它递归的生成UI树，如此一来就无需使用VueRender（不会创建wrap），也无需手动维护父子组件关系了。
  5. NodeStore，提供垃圾回收机制
  6. 考虑使用指令封装dom操作。
