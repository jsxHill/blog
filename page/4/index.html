homebrew使用指南

homebrew是什么

mac上的软件包管理器。

安装与使用

一开始在使用brew时，会遇到权限问题。因为brew install需要向系统目录/usr/local写入文件，而普通操作用户是没有相关权限的。
High Sierra之前的版本可以通过sudo chown -R $(whoami) /usr/local解决。
之后的版本因为Mac系统的限制，/usr/local无法被直接chown，权限问题只能通过卸载/重装homebrew解决（参考cant-chown-usr-local-for-homebrew-in-osx-10-13-high-sierra）。

使用国内镜像

homebrew资源有两种，一是位于github上的repo，二是位于bintray上的二进制bottles。国内访问这两个站点都是非常慢的，因此最好替换为国内镜像。

配置可以参考清华大学Homebrew镜像使用帮助。

brew cast

brew cast主要用来安装一些带界面的应用软件，类似安装App应用。

babel7核心概念

什么是babel

babel是一个能将es6+代码转换为功能等价的es5代码的工具。使得开发人员即使工作在较老的js环境，也能享受到新特性带来的好处。也可以让开发者提前体验实验特性（未加入到浏览器）。

配置

babel具体的语法转换功能是由插件实现的，一种插件对应一种语法转换。比如@babel/plugin-transform-arrow-functions负责将箭头函数转为es5函数：

CLI配置

通过cmd options传递配置项。

文件配置

babel支持多种配置文件：
.babelrc 静态配置
.babelrc.js 支持编程
babel.config.js
babel.config.json
babel.config.json5
babel.config.cjs 模块模式下使用

项目级配置

自babel7开始，支持在项目根路径下添加配置文件作为projectConfig。projectConfig对该项目下的所有babel作业有效，但是它的优先级较低，可以视为defaultConfig。

文件级配置

文件级配置的查找逻辑是：

  1. 从filename选项所在目录开始查找.babelrc文件。
  2. 如果所在目录没有，则继续向上查找.babelrc文件。
  3. 如果遇到包含package.json文件或者超出babelrcRoot指定的目录范围，则停止查找。
    如果存在文件级配置，那么它会跟projectConfig合并，生成新的配置对象，从而实现不同的子目录拥有不同的babel配置。

API配置

以编程的方式传递配置项（优先级高于文件配置）。

polyfill

babel不仅仅要解决语法转换问题（比如class关键字），还要解决jsApi兼容问题（比如IE浏览器不支持Promise对象）。polyfill的作用就是为当前环境打补丁，使其支持不兼容的jsApi。

babel6以前是使用@babel/polyfill，其内部会使用core-js2（core-js是专注于es6+polyfill的库）。
babel7以后做了优化，最好直接引用core-js3（@babel/polyfill与core-js3不兼容），原因如下：
1.core-js2已停止维护，部分以及未来的es6+特性只有core-js3才支持。
2.core-js3修复了一些历史bug，移除了一些已被废弃的api。
3.core-js3分包更灵活（具体可参阅core-js@3, babel and a look into the future），配合babel可以加载更小的包。

值得一提的是，尽管core-js3按需加载做的很强大，但是其体积仍然不容小觑。以常见的Vue + Webpack4项目为例，如果要兼容IE，core-js大概在50kb（minify）左右。

babel-plugin-transform-runtime

babel polyfill可以解决api兼容性问题，但是它也“污染”了全局环境（修改built-ins对象、添加新的全局对象等等）。如果你无法掌控环境的话（比如开发一个库），这可能是个问题。[babel-plugin-transform-runtime][#babel-plugin-transform-runtime]能够帮助你创建一个沙盒环境，从而允许局部引入Symbol、Promise、Set、Generator等es6+新特性。

除了es6+新特性之外,plugin-transform-runtime还可以将babel内部使用的helper函数抽离到一个单独的module。跟默认的inline转换相比，可以节省一些代码体积。

总结一下，babel-plugin-transform-runtime提供3种转换（都不会污染全局环境）：

  1. regenerator aliasing，支持生成器函数；
  2. corejs aliasing，支持其它es6+特性，比如Promise/Symbol/Set（需开启corejs选项并安装对应的@babel/runtime-corejsX）；
  3. helper aliasing，抽离helper函数到独立的module；

preset

preset是babel预设的插件集合，设置preset就相当于引入该preset下的所有插件。
常见的presets有es2015、stage-0、stage-1、stage-2、stage-3、env等等。每类preset都有自己的插件筛选逻辑，比如es2015包含所有es6特性；stage-3包含TC39中的Candidate类特性（ps:stage-x是半开区间。比如stage-1，除了自身还包含所有来自stage-2、stage-3的插件）。
env比较特殊下面会单独叙述。

babel-preset-env

随着js运行时的发展，越来越多的es6+特性得到原生支持，反过来导致很多在preset中预设的插件是不必要的。因此就出现了evn，一种根据目标环境选择插件的preset。它的目的就是尽量只提供必需的语法转换（esX、stage-x等preset已过时）。
env实现的原理是通过收集js-runtime的兼容数据（比如compat-table-es6）获知各环境所缺少的特性，再结合browserslist确定目标环境，最后得出当前需要加载的插件列表。

useBuiltIns

preset.env.useBuiltIns选项告诉babel如何处理polyfill。

useBuiltIns:entry 代表在程序入口处自动拆分core-js或者@babel/polyfill:

useBuiltIns:usage 代表在每个文件的使用处自动注入polyfill：

useBuiltIns:false 不使用自动polyfill。

babel-minify（Babili）

前端工程的最后一步往往是minify bundle（压缩代码）。babel-minify就是配合babel使用的压缩插件集合。
与Uglify等压缩工具相比，babel-minify的优势在于：

  1. 理解es6+语法，可以直接压缩es6+代码（Uglify只支持es5），有助于减少最终体积（无需转译es6代码的场景下）。
  2. 插件化，可定制压缩功能。
  3. 可以结合Flow/TypeScript注释实现更多高级压缩。
    对个人而言，babel-minify最吸引人的就是，它不仅仅是移除空白、注释，缩短变量名，而且还能扩展以支持高级压缩。
    比如，通过语法分析删除dead code： 统一表达式以获得更好的压缩性： 甚至通过constant-folding等手段提升运行时性能：

verdaccio自身的问题（至少3.x版本存在）。

solution:
任选其一

  1. 在verdaccio web站点上登录一次之后，即可通过npm cli登录。
  2. 将max_users设为1，让verdaccio允许登录，同时无法在客户端注册（因为我们至少内置了一个账号）。

vue2底层机制解析

reactive data

@1 响应式原理包含3个部分：

  1. Observer，底层观察者
  2. Dep，订阅列表
  3. Watcher，高层订阅者

reactive obj/array
Observer将一个对象（数组）变为“可响应式的”。

reactive obj会将所有属性会变为可响应式属性，
reactive array会将所有子元素变为可观察的。

reactive prop（可响应式属性）
可响应式属性是利用 Object.defineProperty 将原属性转换为getter/setter，并且绑定一个Dep，让自己的变化可追踪。
除了实现原本的get/set功能之外，getter还会负责收集依赖（depend）。而setter还会负责将新value变为可观察的。如果属性值发生了变化，还会负责
发送通知（Dep.notify）。

reactive method（可响应式方法）
响应式数组的'push','pop','shift','unshift','splice','sort','reverse'也是响应式的。并且还会负责将新元素（有的话）也变为可观察的。

Dep
每个reactive obj以及reactive prop都会绑定一个dep对象，用于记录其订阅者。
订阅的过程称之为收集依赖（depend）。depend是一个全局操作，每次只有一个watcher（target）可以depend。

Watcher
订阅响应式事件。
响应式属性与watcher是多对多关系。
vm实例与watcher是一对多关系。

component

Vue将可配置的实例属性都放在了vm.options下，且vm.options由class options 以及 context options混合而成；
其余实例属性都以_开头，作为私有属性供内部使用。

@1 strategies option合并策略
{
// 先检查合并是否发生在实例上，然后调用 defaultStrat
‘el’: validDefaultStrat,
‘propsData’: validDefaultStrat,

// 会将所有hook并入一个数组，[...parent, ...child]
'beforeCreate': mergeHook,
'created ': mergeHook,
'beforeMount': mergeHook,
'mounted': mergeHook,
'beforeUpdate': mergeHook,
'updated': mergeHook,
'beforeDestroy': mergeHook,
'destroyed': mergeHook,
'activated': mergeHook,
'deactivated': mergeHook,
'errorCaptured': mergeHook,

// parent以原型代理方式并入child, child -> parent
'component': mergeAssets,
'directive': mergeAssets,
'filter': mergeAssets,

// 与mergeHook类似，相同的项使用concat合并，{ _path: [...parent, ...child] }
'watch': mergewatch,

'data': mergeData,

// 浅复制，child覆盖parent

'methods': mergeObj,

'provide': mergeDataOrFn,

'*': defaultStrat

}

@2 mergeOption是创建Vue实例的核心步骤。
每个实例的$options都来自 ConstructorOptions < InstanceOptions 的合并。
而 ConstructorOptions 是来自 OwnConstructorOptions < SuperConstructorOptions … < VueConstructorOptions 的合并。

Vue.extend（继承）就是通过 ConstructorOptions 的逐级合并而实现的。
Sub.options = mergeOptions(Sub.superOptions, extendOptions);

另外还会缓存3个对象，分别是：
1. 父级options，Sub.superOptions = Super.options;
2. 本级options Sub.extendOptions = extendOptions;
3. 原始构造函数options（只读），Sub.sealedOptions = extend({}, Sub.options);
主要目的是为了优化Vue.mixin。mixin动态修改options的原理是：
每当创建Vue实例时，会重新计算一次ConstructorOptions（resolveConstructorOptions），以保证后续动态添加的mixin会包含在其中。
但是全属性的mergeOptions是一件很耗时的任务，因此利用缓存做对比以及局部合并，尽量缩小合并范围，提升性能。

各生命周期钩子触发的时机
beforeCreate Vue实例创建时vue.init内触发；此时options已成形，但是还未进入数据观察状态；
created vue.init末尾触发，实例已进入观测状态，但是还未进行模版渲染；
beforeMount vue.$mount开始时触发，此时render函数已编译好，但是还未生成VNode；
mounted vue.$mount结束时触发，此时VNode已生成完毕，并且已经渲染到为DOM；
beforeUpdate vue._update开始时触发（首次渲染不会触发），局部DOM还未更新；
updated flushSchedulerQueue结束时触发，每个watcher都会触发一次，局部DOM已更新；
activated flushSchedulerQueue结束时触发，keep-alive组件专属；
deactivated keep-alive组件在destory结束时触发；
beforeDestroy 非keep-alive组件在destory开始时触发，此时实例仍可用；
destroyed $destory结束时触发，此时实例已解除所有绑定和事件监听，子组件实例也已经销毁；

weapp包管理

使用npm

前提条件

  1. 小程序基础库版本 2.2.1 或以上（依赖客户端环境）。
  2. 开发者工具 1.02.1808300 或以上开始。
    开发时，仍然通过npm管理package。dependencies字段下的pkg会被打包进入小程序，devDependencies可使用但不会参与构建。
    官方文档

ps：project.config.js可通过miniprogramRoot字段指定node_modules目录。

引用npm包

纯js包直接通过require引用。

自定义组件需要在json中配置。

小程序npm包

因为小程序的运行环境比较特殊，一些诸如window、document、Function、nodejs内置库等等对象并不能使用，所以一般的npm package，除非是纯js库，并不能直接拿到小程序中使用。为了规范书写符合小程序运行条件的包，因此制定了相关约束：

  1. 小程序npm包根目录下必须有构建目录miniprogram_dist（也可以通过package.json.miniprogram指定其它目录）。
  2. 只有miniprogram_dist目录下的文件才会被小程序编译。
  3. package.json中devDependencies字段引用的pkg不会打包到小程序包中。

npm构建

node_modules目录并不会直接参与小程序编译，而是需要走小程序的npm构建流程：

  1. 在项目配置中勾选启用npm模块。
  2. 点击工具->构建npm。
  3. 之后会在node_modules同级目录下生成miniprogram_npm目录。
  4. 搜索源码中所引用的npm包。
  5. 如果是标准小程序包，则将其miniprogram_dist下的文件Copy到miniprogram_npm。
  6. 如果是纯js包则从入口文件（main或index.js）开始，走一遍依赖分析和打包流程（类似webpack），最后将打包后的文件拷贝至miniprogram_npm。

miniprogram_npm目录下才是小程序真正使用的npm包。

发布小程序npm包

miniprogram-cli可以协助生成以下项目模版：

  • 第三方自定义组件（custom-component）
  • 小程序 quickstart（miniprogram）
  • 小程序插件 quickstart（plugin）
  • 小游戏 quickstart（game）

发布纯js包

可以参考sm-crypto模块。

anaconda

anaconda是什么

anaconda是世上最流行的科学计算python发行版之一。它提供了大量与科学计算相关的库以及包管理功能，并且支持虚拟环境(virtual environment)隔离。能够帮助使用者更加安全快速的搭建科学计算类python项目。

channel

channel就是conda用来存放pkg的仓库。

conda-forge是一个比默认仓库更稳定、包更多的仓库。

为channel设置优先级的目的，是为了统一从conda-forge安装包，如果找不到才尝试从其它channel（如defaults）安装。避免多channel带来的版本依赖冲突问题。

conda与pip

pip同样也是python的包管理器，但是混用conda与pip的话，容易造成环境的不稳定。因为conda无法察觉pip对环境所做的修改，一些包会“意外”升级，导致难以排查的版本兼容问题，反之亦然。因此在试图使用pip时，最好遵循以下原则：

  1. 如果可以，只使用conda。
  2. 如果必须使用pip，那么为此新建一个env，以达到隔离效果。
  3. 混合模式下，先使用conda，后使用pip。
  4. 混合模式下，如果需要变更，创建一个新的env来实施。
  5. 导出混合模式的依赖文件，便于导入新env。

miniconda

miniconda是ananconda的最小化版本，默认只包含少量的pkg，没有GUI界面，命令行与完整版一致。适合将conda只视为python包管理工具使用。

核心对象

Spider

scrapy.Spider用于定义如何抓取一个网站的内容。基本流程为：
rules filter -> start requests -> response parse -> extract data -> follow
我们需要做的，就是为每个部分填充具体的业务逻辑。

通过命令行为Spider传递参数

由-a 传递的参数会传递给Spider的__init__方法，并成为spider对象的属性。

Selector

Selector用于使用XPath或者CssSelector从html字符串中查询数据。

Selector内部依赖于parsel，而parsel依赖于lxml。与原库相比，Selector功能更加丰富：

  1. 扩展了Css选择器，支持::attr()匹配属性值，::text匹配文本节点
  2. 扩展了XPath，支持正则式、Set
  3. 支持正则式筛选处理搜索结果，如selector.xpath(“path”).re(r’xxx’)
  4. 链式混用xpath与css，充分发挥2者的优势，如selector.css(“.cls”).xpath(“//a/@href”).get()

Item

Item用于定义ExtractData的数据结构，让提取逻辑更加清晰。
开发者一般会使用自由度较高的dict来保存提取结果。但dict对象自身结构不受限制，可以任意修改。而且除了翻阅源代码，只能通过文档方式来描述现有dict的结构，在复杂场景下，这往往会引起代码可维护性问题。Item正好提供了一个集中描述结构化数据的地方。

Item是装载数据的容器，Item Loader则是专门为容器填充数据的工具。Item本身只能通过item[key]=value形式去操作数据，ItemLoader则允许你定义更加丰富的行为。
比如：

  1. 支持为field添加多个数据来源，组合输出。
  2. 支持input-processor，可以进行简单的数据格式化。
  3. 支持output-processor，自定义数据来源输出结果。

Item Pipline
ItemPipline用于流式处理Item。可以用来：

  1. 数据清洗
  2. 数据校验
  3. 数据筛选
  4. 数据存储
    ps: 可以通过no return或者raise DropItem(“reason”)丢弃Item，被丢弃的Item也不会本后续的流处理。

Feed Exports

Feed Exports用于以文件形式导出数据。支持JSON、JSON Lines、CSV、XML等多种文件格式，以及本地文件、FTP、S3等多种存储方式。
简单场景下，可以通过命令行参数 -o 直接导出数据。复杂场景下，可以自定义ItemExport，结合ItemPipline流程序列化存储数据。

Requests and Responses

Requests-Responsesrequest用于发送http请求，responses用于解析html文档。

Link Extractors

Link Extractors用于导出页面中的link对象，便于实现链接追踪。

js函数式编程

核心概念

-> Programming
-> lambda calculus lambda计算（自上而下的编程方式，函数式）
-> 函数匿名
-> 只接受一元参数
-> 函数为一等公民
-> Turing machine 图灵机（自下而上的编程方式，指令式）
-> 两者在数学上是等价的，任何图灵机能实现的程序，都可以转化为lambda计算

-> Functino Programming 函数式编程
-> non-determinism（不确定性） = parallel processing（并行程序） + mutable state（修改状态）
-> Pure Function（纯函数）：
-> 只用于Mapping（映射）计算的函数
-> 无副作用（一个input恒对应一个output，也不会修改或依赖任何外部变量）
-> Immutability 不可变性
-> 不修改任何外部变量
-> 不共享任何state
-> 有一个纯函数特例noop（无返回值）

-> Function Composition 函数组合
-> f∘g <=> (f∘g)(x) <=> f(g(x))
-> 函数组合是fp的核心
-> curry（柯里化）：
-> 一次只处理一个参数，然后返回处理下一个参数的函数，直到最后一个参数为止
-> 柯里化的函数是抽象的（Abstraction），偏函数是它的实例
-> 3种fp：
-> Functions map: a => b
-> Functors map with context: Functor(a) => Functor(b)
-> Monads flatten and map with context: Monad(Monad(a)) => Monad(b)

-> Functor Map 函子运算
-> functor是一种抽象模型
-> mappable，可通过map接口将范畴A映射为范畴B
-> Identity，恒等，f.map(x => x)的结构在含义上应该与f相等
-> Composition，组合，f.map(x => h(g(x)))等价于f.map(g).map(h)
-> 利用函子的组合特性，可方便实现函数组合的链式调用
-> functor的意义在于提供一种统一的方式去处理任意数据

-> Monad Map Monad运算（源自数学之范畴学）
-> monad是一种增强的functor，对外提供flatMap功能
-> flatMap
flatten M(a) -> a
f a -> b
lift M(b)
-> monad的目的在于为value添加上下文信息（Context）
-> 空值处理
-> 异常处理
-> 异步处理

-> MOP 面向组件编程，用数学替代inheritance
-> OO的本质
-> Encapsulation，封装状态
-> Message passing，消息通信
-> Dynamic binding，动态绑定
-> Object Composition 对象组合
-> 组合优于继承，继承是一种耦合度很高的对象复用方式，is-a会带来诸多问题：
-> 脆弱的基类——因为会影响诸多class，导致基类很难修改
-> 大猩猩香蕉问题——继承可能会带来一堆不需要的属性或者方法
-> 必然的重复——不同子类可能会包含相同实现，因为并非所有接口实现都不同
-> 常见的对象组合方式：
-> Aggregation 聚集，由一组可枚举的对象组成
-> Concatenation 串联，向一个对象上动态添加新属性
-> Delegation 委托，对象将访问转发给另一个对象
-> Injection 注入，对象作为参数传递给其它对象

-> Functional vs Class
-> 优先使用函数
-> 对于js而言，class是最难使用组合模式的
-> 组合复杂度：Functions > objects > factory functions > functional mixins > classes
-> class extends会带来is-a问题
-> 避免使用new
-> new class会被迫与Constructor函数耦合，重构代码变得困难
-> new class会隐式告诉别人这是一个构造函数，构造函数可以extends，而extends会导致脆弱的代码结构出现
-> 对外暴露factory function，而非class
-> class只能通过new来创建对象
-> Class适用场景
-> 封装底层通用框架
-> 此时业务比较封闭，可以设计出健康的层次结构
-> 统一创建组件实例以及管理其生命周期，对外隐藏组件class
-> 注意class最多extends一层
-> 微观性能优化
-> 利用prototype节约内存
-> 大多数场景可以忽略不计。因此只在真的有必要时考虑prototype。
-> 比如react、vue等UI组件库
-> 比如three.js等图像引擎，他们对实时渲染性能要求较高

-> Lens 一种对属性取值以设值的抽象模型
-> lens提供2个纯函数来完成取值以及设值
-> 范式
-> view(lens, set(lens, a, store)) ≡ a
-> set(lens, b, set(lens, a, store)) ≡ set(lens, b, store)
-> set(lens, view(lens, store), store) ≡ store

-> Transducers 转换器
-> Transducers是高阶reducer（缩减器）
-> 函数组合reducer
-> 高效的流式数据处理
-> 能够处理任意结构的可枚举对象
-> 无论是lazy（懒求值）或者eager（及早求值）都不影响Transducers
-> 签名
-> reducer = (accumulator, current) => accumulator
transducer = reducer => reducer
即
transducer = ((accumulator, current) => accumulator) => ((accumulator, current) => accumulator)
-> reducer是用于将多个输入变为一个输入的函数
-> Top to bottom / Left to right
-> Function Compose f∘g∘h => reducer(h) -> reducer(g) -> reducer(f)
-> rules
-> Initialization，如果accumulator没有初始值，那么transducer必须调用step为其生成一个empty值（valid initial accumulator）
-> Early termination，当接收到一个reduced时，transducer必须检查并停止pipe（提前终止reducing）
-> Completion(optional)，当缩减过程结束时，可以选择性执行一个Complete操作（after reducing，do sth）
-> fp编程环境
-> 使用js框架：Ramda RxJS Mori
-> 与underscore、lodash相比，前者是js版函数式编程，而后者是命令式的工具方法（如今很多方法都可以用原生方法替代）
-> Clojure（Script）从Lisp发展而来的函数式编程语言
-> ClojureScript可以将Clojure编译为js。如此一来，Clojure代码可以调用js函数以及在browser/node中运行
